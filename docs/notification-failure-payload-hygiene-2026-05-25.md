# Notification Failure Payload Hygiene - 2026-05-25

## Scope

- Twilio SMS and voice failure alerts routed through `server/services/twilio-failure-alerts.js`.
- Admin notification payloads and metadata generated by the `twilio_failure` trigger.

## Guardrail

No customer SMS, customer email, or live voice calls were sent during this audit. Production verification was limited to read-only health and Railway log checks.

## Findings

- Twilio failure logs masked phone numbers and SIDs, but the admin notification payload could still persist raw provider error text.
- The persisted notification metadata included a raw Twilio SID field and a dedupe key that could include full SIDs or phone numbers.

## Remediation

- Sanitized Twilio provider failure text before it reaches admin notification body or metadata.
- Removed raw SID from `twilio_failure` notification payloads; only masked SID is sent.
- Replaced persisted dedupe keys with stable SHA-256-derived public keys.
- Added focused regression coverage for phone, email, URL, SID, and dedupe-key hygiene.
